# Fluxo Cliente → Prestador

Este documento descreve o fluxo da plataforma no qual o cliente cria uma solicitação para um serviço que ainda não está disponível na plataforma, permitindo que prestadores interessados analisem a demanda e apresentem propostas.

## 1. Criação da Solicitação

* O cliente autenticado cria uma solicitação de serviço.
* A solicitação registra informações como:

  * título;
  * descrição;
  * categoria;
  * observações adicionais.
* Após o cadastro, a solicitação fica disponível para visualização pelos prestadores.

⬇️

## 2. Análise da Solicitação pelo Prestador

O prestador visualiza as solicitações disponíveis e pode:

### ❌ Recusar

* O processo é encerrado para aquele prestador.
* Nenhum orçamento será gerado.

### ✅ Aceitar

* O prestador demonstra interesse em atender a solicitação.
* O fluxo segue para a geração do orçamento.

⬇️

## 3. Geração do Orçamento

* O prestador cria um orçamento para a solicitação aceita.
* O orçamento registra informações como:

  * valor;
  * descrição;
  * observações adicionais.
* O orçamento é enviado ao cliente para análise.

⬇️

## 4. Análise do Orçamento pelo Cliente

O cliente recebe o orçamento e pode:

### ❌ Recusar

* O processo é encerrado.
* O serviço não será executado.

### ✅ Aceitar

* O orçamento é aprovado.
* O fluxo segue para a criação do agendamento.

⬇️

## 5. Criação do Agendamento

* O cliente cria um agendamento para execução do serviço.
* Durante esse processo, informa:

  * data;
  * horário;
  * observações adicionais.

⬇️

## 6. Análise do Agendamento pelo Prestador

O prestador recebe a solicitação de agendamento e pode:

### ❌ Recusar

* O processo é encerrado.
* O serviço não será executado.

### ✅ Aceitar

* O agendamento é confirmado.
* O fluxo segue para execução do serviço.

⬇️

## 7. Execução do Serviço

* O prestador realiza o serviço na data e horário definidos.
* A conclusão da atividade confirma a entrega do trabalho contratado.

⬇️

## 8. Avaliação do Serviço

* Após a conclusão, o cliente pode registrar uma avaliação.
* A avaliação contém:

  * nota;
  * comentário;
  * vínculo com o serviço executado.

---

## Resumo do Fluxo

1. Cliente cria uma solicitação de serviço.
2. Prestador aceita ou recusa a solicitação.
3. Prestador gera um orçamento.
4. Cliente aceita ou recusa o orçamento.
5. Cliente cria um agendamento.
6. Prestador aceita ou recusa o agendamento.
7. Prestador executa o serviço.
8. Cliente avalia o serviço.

> Fluxo Cliente → Prestador: o cliente publica uma necessidade de serviço na plataforma, o prestador apresenta uma proposta por meio de orçamento e, após as aprovações necessárias, o serviço é executado e avaliado.
